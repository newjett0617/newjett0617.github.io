[Docker] 使用 Traefik 來當作反向代理 Reverse Proxy

最近在伺服器架設了一個應用程式，這個應用程式是跑在 Docker 上，假如應用程式需要對外服務的話，通常會需要暴露一個 port 才能將請求轉導到容器內，可以透過 curl http://xx.xx.xx.xx:8080 可以訪問到容器的 nginx 應用程式

docker run -d --publish='8080:80' nginx:alpine

一但暴露了 port 話，任何人可以用掃描工具發現這個 IP 的 port 是開啟的狀態（IPv4 跟 port 是有範圍1），就有機會對 port 做一些事情

nc -v -z xx.xx.xx.xx 8080

Connection to 172.104.85.116 8080 port [tcp/http-alt] succeeded!

通常會限定是只有 127.0.0.1 的才能訪問該 port，這樣的話外部就不會發現 8080 port 是被打開的

docker run -d --publish='127.0.0.1:8080:80' nginx:alpine

CONTAINER ID   IMAGE          COMMAND                  CREATED              STATUS              PORTS                    NAMES
05179d024297   nginx:alpine   "/docker-entrypoint.…"   About a minute ago   Up About a minute   127.0.0.1:8080->80/tcp   nginx

nc -v -z xx.xx.xx.xx 8080

nc: connect to xx.xx.xx.xx port 8080 (tcp) failed: Connection refused

一但限定只有 127.0.0.1 能訪問的話，外部就無法訪問，必須要有一個反向代理 (Reverse Proxy) 將請求代理到內部

有許多工具可以使用，像是 Apache, Nginx, Caddy 等等都可以做到 reverse proxy 的功能

本文要介紹的 Traefik 跟 Docker 有著密不可分的關係，透過使用 Docker 的物件 labels 來達到目的

只需要在容器貼上標籤 labels，就可以很容易地達到 reverse proxy 目的，Traefik 會自動偵測並接管容器，將請求代理至指定的容器

建置 Traefik

首先需要建立一個給 Traefik 專屬使用的網路

docker network create reverse-proxy

撰寫 docker-compose.yml 後就可以啟動了

version: '3'

networks:
  reverse-proxy:
    external: true

services:
  traefik:
    image: docker.io/library/traefik:v2.3
    container_name: traefik
    restart: always
    command:
      - --api.dashboard=true
      - --providers.docker=true # Enabling docker provider
      - --providers.docker.exposedbydefault=false # Do not expose containers unless explicitly told so
      - --entrypoints.web.address=:80 # Traefik will listen to incoming request on the port 80 (HTTP)
      - --entrypoints.websecure.address=:443 # Traefik will listen to incoming request on the port 443 (HTTPS)
      - --certificatesresolvers.myresolver.acme.httpchallenge=true # Enable a http challenge named "myresolver"
      - --certificatesresolvers.myresolver.acme.httpchallenge.entrypoint=web # Tell it to use our predefined entrypoint named "web"
      - --certificatesresolvers.myresolver.acme.email=postmaster@example.com # The email to provide to let's encrypt
      - --certificatesresolvers.myresolver.acme.storage=/letsencrypt/acme.json # Tell to store the certificate on a path under our volume
    ports:
      - 80:80
      - 443:443
    networks:
      - reverse-proxy
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock:ro
      - ./letsencrypt:/letsencrypt # Create a letsencrypt dir within the folder where the docker-compose file is
    labels:
      - traefik.enable=true
      - traefik.http.routers.traefik.rule=Host(`traefik.example.com`)
      - traefik.http.routers.traefik.entrypoints=websecure
      - traefik.http.routers.traefik.tls.certresolver=myresolver
      - traefik.http.routers.traefik.service=api@internal
      - traefik.http.routers.traefik.middlewares=auth
      - traefik.http.middlewares.auth.basicauth.users=username:password # use basic auth, `$ htpasswd username password` (output replace $ -> $$)
      - traefik.http.services.dashboard.loadbalancer.server.port=8080 # 443 -> 8080

使用 Traefik

在一個 nginx 上加上 label 就能啟動

version: '3'

networks:
  reverse-proxy:
    external: true

services:
  snginx:
    image: docker.io/library/nginx:alpine
    container_name: web
    expose:
      - 80
    networks:
      - reverse-proxy
    labels:
      - traefik.enable=true # Explicitly tell Traefik to expose this container
      - traefik.http.routers.snginx.rule=Host(`snginx.example.com`) # The domain the service will respond to
      - traefik.http.routers.snginx.entrypoints=websecure # Allow request only from the predefined entry point named "websecure"
      - traefik.http.routers.snginx.tls.certresolver=myresolver # Uses the Host rule to define which certificate to issue

可以自己撰寫規則，我是使用 subdomain 方式去匹配 container，也可以使用 path 方式匹配，traefik 提供的功能太多了，以後有碰到再來寫
